How It Works

Understand the data, input format, and forecast output

This page explains what information the forecasting model uses, how the uploaded Excel workbook is interpreted, and what to expect from the generated forecast.

Model Inputs

What data the model uses

Company financial data

The model reads company-level financial statement values from the uploaded workbook. These values form the core financial input used for forecasting.

Financial values extracted

  • Revenue
  • Gross Profit
  • Interest Expense
  • Operating Expense
  • Tax
  • Net Income

Industry trend data

The system also adds industry-level context using an internal library of IBIS industry files. This gives the model information about the wider business environment surrounding the company.

Industry values extracted

  • Revenue Change Percentage
  • Employment Change Percentage
  • Wages Change Percentage

How this file is selected

After the workbook is uploaded, the system reads the company’s Primary SIC code, maps it to the relevant NAICS industry, searches the internal IBIS library for matching files, and selects the industry file associated with that company and year.

Single workbook requirement

The system expects a single Excel workbook that contains all required data in one file. This includes company information, income statement data, and the broader company report structure used by the parser.

The uploaded Excel file should contain:

  • Company information section, including the company name and Primary SIC
  • Income statement section with yearly financial data
  • Balance sheet section as part of the same workbook structure

Important note

All of this information should exist within the same Excel file. The system scans the workbook to locate these sections automatically, so separating them into multiple files will not work correctly.

Sample files

Sample input files will be available on the Try It page so users can test the model and understand the expected workbook format.

What a valid input should include

01

One Excel workbook

Upload a single Excel file that contains the company information section, income statement, and balance sheet structure together.

02

Company identifier

The workbook should contain the company name and a Primary SIC code so the system can map the company to the correct industry context.

03

Report dates and labels

The spreadsheet should include yearly report dates and recognizable financial labels so the parser can locate the required values.

04

Usable base year

The base year selected by the user must exist in the uploaded data so the model has a valid starting point for forecasting.

Forecast Process

What happens after you upload a file

The application parses the workbook, extracts company metrics, adds industry context, and then generates year-by-year forecasts.

01

Parse the workbook

The uploaded spreadsheet is scanned to identify company details, reporting years, and the sections containing financial statement data.

02

Extract company metrics

The system pulls the financial values needed for forecasting, including revenue, gross profit, expenses, tax, and net income.

03

Add industry context

Using the company’s Primary SIC code, the system maps to the relevant industry file and adds revenue, employment, and wages trend values.

04

Generate forecasts

A deep neural network predicts the next year’s financial values, then repeats that process recursively for longer forecast ranges.

What to Expect

What the output means

After a successful upload, the system generates forecasted financial values for each year from the selected base year up to the selected target year.

The output is not a simple label or score. Instead, the model produces projected numerical values for revenue, gross profit, interest expense, operating expense, tax, and net income.

For multi-year predictions, the system works recursively. Each predicted year becomes the input for forecasting the next one, creating a year-by-year projected sequence.

Forecasts should be understood as model-based projections. The quality of the result depends on the structure, completeness, and consistency of the uploaded workbook.
